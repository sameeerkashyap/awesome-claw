# 🤝 Contributing to Awesome Claw

Thank you for your interest in contributing to **Awesome Claw**! This curated list relies on community participation to stay comprehensive and up-to-date.

## 📋 Guidelines

### Adding a New Project

To add a new OpenClaw-inspired project, please submit a pull request with the following:

1. **Update the Comparison Table** in `README.md` with a new row including:
   - Project name (linked to repo)
   - Language
   - Memory footprint
   - Startup time
   - Supported channels
   - Architecture style
   - Security model

2. **Add a Detailed Section** following the existing format:
   - Project name and emoji
   - Blockquote tagline
   - Info table (repository, language, license, contributors)
   - 1–2 paragraph description
   - Key highlights as a bulleted list with emojis

### Criteria for Inclusion

Projects should meet the following criteria to be listed:

- ✅ **Open source** — publicly available source code
- ✅ **OpenClaw-inspired** — clearly part of the Claw ecosystem or inspired by it
- ✅ **Active development** — recent commits and maintained codebase
- ✅ **Functional** — the project should be usable, not just a skeleton or placeholder
- ✅ **Documentation** — has a README with installation and usage instructions

### Updating Existing Entries

If a listed project has significantly changed (new features, renamed, archived), please submit a PR to update the relevant information.

### Reporting Issues

- 🐛 **Broken links** — open an issue if any link is dead or redirects
- 📝 **Inaccurate info** — let us know if a description is outdated
- 💡 **Suggestions** — ideas for new categories or improvements are welcome

## 🔀 Pull Request Process

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b add-project-name`)
3. **Make your changes** following the format and style of existing entries
4. **Commit** with a clear message (`git commit -m "Add ProjectName to list"`)
5. **Push** to your fork and **open a pull request**
6. Provide a brief description of the project and why it fits the list

## 🎨 Style Guide

- Use **consistent formatting** — follow the existing markdown structure
- Use **emoji prefixes** for highlights (🔒, ⚡, 🌍, etc.)
- Keep descriptions **concise but informative**
- Use **dynamic badges** for star counts (see existing entries for the format)
- Link to the **repository**, not external marketing pages
- Use tables for metadata (repository, language, license, contributors)

## 📜 Code of Conduct

Be respectful, constructive, and inclusive in all discussions and contributions. We're all here to support the open-source AI assistant ecosystem.
